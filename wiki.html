Общее описание

Sputnik Studio — профессиональное веб-приложение для инженерного проектирования аудиовизуальных систем, систем видеоконференцсвязи, LED-дисплеев, сетевой инфраструктуры и эргономики рабочих мест. Инструмент позволяет выполнять комплексные расчёты, сохранять проекты в браузере, экспортировать их в JSON и формировать печатные отчёты.

Приложение построено на модульной архитектуре: каждый раздел (видео, сеть, LED, звук, ВКС, тракты, эргономика) содержит независимые калькуляторы, которые обмениваются данными через общее хранилище. Это позволяет создавать сквозные проекты, где параметры одного раздела автоматически влияют на смежные расчёты.

Интерфейс адаптирован для работы на настольных компьютерах, планшетах и смартфонах. Доступна тёмная тема, сворачиваемая боковая панель, плавная навигация по разделам. Все вычисления производятся на стороне клиента, данные не покидают устройство пользователя.

Основные возможности:

  • Расчёт пропускной способности видеопотоков с учётом кодека, разрешения, частоты кадров, цветовой субдискретизации и битности.
  • Моделирование сетевой нагрузки с поддержкой QoS, Multicast, PoE и расчётом суммарного битрейта.
  • Проектирование LED-экранов в трёх режимах: по кабинетам, по разрешению, сшивка экранов. Автоматический расчёт физических размеров, разрешения, площади, мощности и минимального расстояния просмотра.
  • Акустические расчёты: уровень звукового давления (SPL), время реверберации (RT60) по формуле Сэбина, индекс разборчивости речи (STI).
  • Оценка требований к видеоконференцсвязи (битрейт для P2P и MCU-конференций) с учётом популярных платформ и кодеков.
  • Построение сквозных трактов сигнала с учётом задержек, PoE-бюджета и энергопотребления устройств.
  • Эргономический анализ зон видимости экранов коллективного пользования по методике угловых размеров (α = 0.25°, 0.38°, 0.55°).
  • Управление проектом: автосохранение в localStorage, экспорт/импорт JSON, печать отчёта, сброс данных.

Необходимая минимальная квалификация инженера

Для эффективного использования Sputnik Studio и корректной интерпретации результатов расчётов специалист должен обладать следующими компетенциями:

  • Видеотехника: глубокое понимание цифровых видеопотоков (разрешения, кадровая частота, цветовые пространства RGB/YCbCr, субдискретизация 4:4:4/4:2:2/4:2:0, битность 8/10/12). Знание кодеков H.264, H.265, MJPEG, SDVoE, NDI, их областей применения и влияния на задержку и битрейт. Умение рассчитывать несжатый битрейт по формуле B = W×H×FPS×bitDepth×channels.
  • Сетевые технологии: знание стандартов Ethernet (10/100/1000/10G), принципов коммутации, VLAN (802.1Q), IGMP Snooping для оптимизации Multicast, QoS (DiffServ, 802.1p), PoE (802.3af/at/bt) с расчётом бюджетов, протоколов синхронизации PTP (IEEE 1588) и NTP. Опыт настройки управляемых коммутаторов для AV-over-IP.
  • LED-дисплеи: понимание шага пикселя (pitch), типов кабинетов, расчёта физических размеров и разрешения, требований к минимальному расстоянию просмотра (L_min = 1000 × pitch (мм)), оценки мощности и тепловыделения, калибровки яркости и цвета.
  • Акустика и звукоусиление: знание логарифмической шкалы (дБ), расчёта SPL (SPL = Sens + 10·log(P) - 20·log(D)), времени реверберации по Сэбину (RT60 = 0.161·V / Σ(αᵢ·Sᵢ)), индекса разборчивости STI, подбора громкоговорителей по диаграмме направленности.
  • Видеоконференцсвязь: понимание протоколов SIP/H.323, архитектур MCU и SFU, расчёта битрейта для P2P и многоточечных конференций, учёта задержек и джиттера, настройки QoS для приоритизации RTP-трафика.
  • Эргономика: знание угловых порогов зрения человека (угловая минута), методов расчёта оптимальной зоны видимости (ЗОВ) и коэффициента эргономики Кэ, требований к размерам шрифтов и дистанциям до экранов.
  • Нормативная база: умение применять ГОСТ Р, SMPTE, IEEE, ISO, AVIXA, BICSI в проектной документации.
  • Проектирование: опыт создания технических заданий, однолинейных схем, спецификаций оборудования, расчёта бюджетов (мощность, охлаждение, ИБП).

Методология расчётов

Все расчёты в Sputnik Studio основаны на признанных инженерных формулах, аппроксимациях, полученных из практических измерений, и отраслевых стандартах. Ниже приведены ключевые методики, используемые в приложении.

Видеопотоки

Битрейт несжатого видео (Мбит/с) = (ширина × высота × FPS × битность × кол-во каналов) / 1 000 000

Для сжатых видеопотоков применяются эмпирические коэффициенты, основанные на типовых профилях кодеков:

  • H.264 (High Profile): 1080p@30 — 2–4 Мбит/с, 4K@30 — 8–15 Мбит/с.
  • H.265 (Main Profile): на 30–50% ниже H.264 при том же качестве.
  • MJPEG: битрейт = (ширина × высота × FPS × 12) / 1 000 000 (приблизительно).

При расчётах учитываются накладные расходы протоколов (RTP, UDP, IP) — добавка 5–10% к полезной нагрузке.

LED-экраны

Разрешение_гор = кол-во_каб_гор × ширина_каб_пикс
Физическая_ширина_м = кол-во_каб_гор × ширина_каб_мм / 1000
Площадь_м² = ширина_м × высота_м
Мощность_Вт = площадь_м² × удельная_мощность_Вт/м²
Минимальное_расстояние_просмотра_м = pitch_мм

Удельная мощность для indoor-экранов принимается 250–300 Вт/м², для outdoor — до 600–800 Вт/м² в зависимости от яркости (5000–10000 нит).

Акустика

SPL (дБ) = Sens + 10·log(P) - 20·log(D) + 10·log(N) (для массива)
RT60 (с) = 0.161·V / Σ(αᵢ·Sᵢ)

Для STI используется модель Шредера, аппроксимирующая связь между RT60 и отношением сигнал/шум. При RT60 > 1.5 с разборчивость резко падает.

Эргономика

h = L·tg(α) или L = h / tg(α)

α — угловой размер символа (градусы):
0.25° (14–16 угловых минут) — минимальная читаемость,
0.38° (20–22 угловых минуты) — оптимальные условия,
0.55° (30–35 угловых минут) — критическая информация.

Для оценки зоны оптимальной видимости (ЗОВ) используется геометрический расчёт углов на каждый символ строки длиной N символов. Коэффициент эргономики Кэ = (средняя доля ЗОВ по всем наблюдателям) × 10.

Глоссарий

SPL (Sound Pressure Level) — уровень звукового давления, логарифмическая мера громкости, дБ.

RT60 (Reverberation Time) — время реверберации, с.

STI (Speech Transmission Index) — индекс передачи речи (0–1), характеризует разборчивость.

PoE (Power over Ethernet) — передача питания по Ethernet: 802.3af (15.4 Вт), 802.3at (30 Вт), 802.3bt (60/90 Вт).

QoS (Quality of Service) — механизм приоритизации трафика (DSCP, CoS).

Multicast — групповая рассылка, оптимизированная IGMP Snooping.

Chroma Subsampling — цветовая субдискретизация (4:4:4, 4:2:2, 4:2:0).

Bit Depth — глубина цвета (8/10/12 бит на канал).

MCU (Multipoint Control Unit) — устройство для многоточечных видеоконференций, микширует потоки.

SFU (Selective Forwarding Unit) — ретранслирует потоки без обработки, снижая нагрузку.

Latency — задержка прохождения сигнала (мс).

HDBaseT — технология передачи видео, аудио, Ethernet, управления и питания по витой паре до 100 м.

PTP (Precision Time Protocol) — протокол точного времени IEEE 1588 для синхронизации.

VLAN (Virtual LAN) — логическая сегментация сети (802.1Q).

IGMP Snooping — механизм коммутатора для управления Multicast-трафиком.

Видео

Видеокодеки

H.264 (AVC) — наиболее распространённый кодек, поддерживаемый всеми устройствами. Профили: Baseline (низкая задержка), Main, High (высокое качество). Для профессиональных систем с низкой задержкой используется Intra-профиль (все кадры — I).

H.265 (HEVC) — более эффективное сжатие, требует больше вычислительных ресурсов. Оптимален для 4K/8K, HDR. Лицензирование через патентные пулы.

MJPEG — покадровое сжатие, минимальная задержка, высокий битрейт. Используется в системах видеонаблюдения и там, где важен каждый кадр.

SDVoE — проприетарный стандарт передачи несжатого видео по 10GbE с задержкой менее 0.1 мс. Требует специализированных коммутаторов и поддержки PTP.

NDI|HX — гибридный кодек (H.264/H.265) для IP-производства, задержка ~25 мс, поддерживает метаданные и управление.

Цветовые параметры

Цветовая субдискретизация (chroma subsampling) обозначается J:a:b:

  • 4:4:4 — полное цветовое разрешение, необходимо для графики, текста, компьютерных интерфейсов.
  • 4:2:2 — цветовая информация дублируется по горизонтали, используется в профессиональном видео.
  • 4:2:0 — дублирование по горизонтали и вертикали, стандарт для вещания, стриминга.

Цветовое пространство: RGB (дисплеи, графика), YCbCr (видео), BT.2020 (UHD/HDR), DCI-P3 (кино).

Глубина цвета (битность): 8 бит (256 градаций), 10 бит (1024), 12 бит (4096). Большая битность необходима для HDR, предотвращает бэндинг (полосатость) на градиентах.

Влияние на битрейт: увеличение битности на 2 бита повышает битрейт несжатого видео примерно на 20%. Для сжатых потоков рост менее значителен из-за адаптивного квантования.

Сеть

При проектировании сетевой инфраструктуры для AV-систем необходимо учитывать следующие аспекты:

  • Среда передачи: Cat5e (1 Гбит/с, 100 м), Cat6/6a (10 Гбит/с, 55/100 м), оптоволокно (многомодовое OM3/OM4 до 300 м, одномодовое до 10 км).
  • Управляемые коммутаторы: обязательны для VLAN, IGMP Snooping, QoS, PoE-бюджетирования, мониторинга.
  • Multicast: снижает нагрузку при передаче одного потока многим получателям. Требует IGMP Snooping на коммутаторах и корректной настройки PIM (для маршрутизации).
  • QoS: приоритизация трафика по меткам DSCP (EF для видео, AF41 для аудио) или CoS (802.1p). Рекомендуется выделять отдельные очереди с минимальными задержками.
  • PoE (Power over Ethernet): расчёт бюджета коммутатора (суммарная мощность всех подключённых устройств не должна превышать PoE-бюджет). Стандарты: 802.3af (15.4 Вт на порт), 802.3at (30 Вт), 802.3bt (60/90 Вт).
  • Синхронизация: для AV-over-IP (Dante, SDVoE, AES67) требуется PTP (IEEE 1588), для систем управления — NTP.
  • Резервирование: критичные системы строятся с использованием протоколов STP/RSTP/MSTP, агрегации каналов (LACP), дублированных коммутаторов и устройств с резервированием питания.

В приложении рассчитывается коэффициент загрузки сети как отношение суммарного битрейта всех видеопотоков (с учётом Multicast) к пропускной способности выбранного коммутатора. Рекомендуемый запас — не менее 20%.

LED-экраны

Светодиодные экраны — ключевой элемент систем визуализации. Sputnik Studio поддерживает три режима расчёта.

Режим 1: По кабинетам

Пользователь выбирает шаг пикселя (pitch) в мм, типоразмер кабинета (ширина×высота) из списка или задаёт свой, затем количество кабинетов по горизонтали и вертикали. Результаты:

  • Физические размеры экрана (ширина, высота) в метрах.
  • Разрешение в пикселях (ширина×высота).
  • Площадь экрана (м²).
  • Ориентировочная потребляемая мощность (Вт).
  • Минимальное расстояние просмотра (м) = pitch (мм).

Режим 2: По разрешению

Задаётся целевое разрешение (Full HD, 4K, 8K или пользовательское). Система подбирает ближайшее целое количество кабинетов, округляя вверх, чтобы обеспечить не меньшее разрешение. Выводится реальное разрешение, физические размеры, количество кабинетов, площадь и мощность.

Режим 3: Сшивка экранов

Позволяет объединить несколько одинаковых LED-экранов в один логический (видеостена). Учитывается количество экранов по горизонтали и вертикали, зазор между кабинетами (обычно 0.5–1 мм). Результат — суммарные размеры, разрешение, площадь.

Дополнительные параметры: яркость (ниты) влияет на мощность и тепловыделение. Для indoor обычно 800–1500 нит, для outdoor — 5000–10000 нит. Для HDR-контента требуется яркость не менее 1000 нит и 10-битная глубина цвета.

Звук

Акустический раздел позволяет оценить параметры звукоусиления и акустики помещений.

Уровень звукового давления (SPL)

SPL (дБ) = Sens + 10·log(P) - 20·log(D) + 10·log(N)

где Sens — чувствительность громкоговорителя (дБ/1Вт/1м), P — подводимая мощность (Вт), D — расстояние (м), N — количество громкоговорителей в массиве.

Время реверберации (RT60)

RT60 = 0.161·V / Σ(αᵢ·Sᵢ)

V — объём помещения (м³), αᵢ — коэффициент звукопоглощения материала i-й поверхности, Sᵢ — площадь (м²). Для конференц-залов оптимальное RT60 = 0.4–0.8 с, для зрительных залов — 1.0–1.5 с.

Индекс разборчивости речи (STI)

STI вычисляется по аппроксимации, учитывающей RT60 и отношение сигнал/шум (SNR). STI ≥ 0.6 считается хорошей разборчивостью, ≥ 0.75 — отличной. При RT60 > 1.5 с разборчивость падает ниже допустимых значений.

Нормативные документы: ГОСТ Р 53578-2009, СП 134.13330.2012, ISO 3382-1.

ВКС (видеоконференции)

Расчёт битрейта для видеоконференций выполняется для двух сценариев: P2P (point-to-point) и многоточечная конференция через MCU или SFU.

P2P-конференция

Каждый участник отправляет один поток (например, 1080p@30) и принимает один поток (также 1080p@30). Суммарный битрейт на стороне клиента = битрейт отправки + битрейт приёма.

Многоточечная конференция (MCU/SFU)

При использовании MCU в режиме коммутации (switch) каждый участник отправляет один поток, но принимает N-1 потоков (по одному от каждого другого участника). Суммарный битрейт приёма = (N-1) × битрейт потока.

При использовании SFU (Selective Forwarding Unit) клиент также получает все отдельные потоки, нагрузка аналогична. В режиме микширования MCU отправляет один смикшированный поток, нагрузка снижается.

В приложении используются аппроксимированные значения битрейта для популярных платформ:

  • TrueConf (H.264): 1080p@30 — 2.5 Мбит/с, 720p@30 — 1.5 Мбит/с.
  • Vinteo IVA (H.265): 1080p@30 — 1.5 Мбит/с, 4K — 5 Мбит/с.
  • Cisco Webex: 1080p@30 — 2.0 Мбит/с.
  • Poly: 1080p@30 — 2.0 Мбит/с.
  • Yealink: 1080p@30 — 1.8 Мбит/с.

Рекомендуется закладывать запас 20–30% на сетевые накладные расходы и флуктуации.

Тракты

Сквозной тракт (сигнальная цепь) — последовательность устройств от источника до приёмника. Позволяет оценить суммарную задержку, энергопотребление, PoE-бюджет и количество задействованных портов.

  • Устройства: источники (ПК, плееры), передатчики (HDMI->HDBaseT), коммутаторы, матрицы, LED-процессоры, экраны.
  • Параметры: задержка (мс) каждого устройства, потребляемая мощность, поддержка PoE (вход/выход), количество портов Ethernet.
  • Расчёт: суммарная задержка (мс и в кадрах при 60 fps), общая мощность, бюджет PoE (если есть PoE-коммутатор).

Тракт позволяет выявить узкие места: превышение допустимой задержки (для интерактивных систем >50 мс нежелательно), перегрузку PoE-коммутатора, недостаток портов. В приложении реализован конструктор тракта с возможностью добавления/удаления устройств и мгновенным пересчётом.

Эргономика ЭКП

Для экранов коллективного пользования (ситуационные центры, диспетчерские) критически важно обеспечить комфортное восприятие информации всеми участниками. В приложении реализованы два калькулятора, основанные на методике угловых размеров.

Калькулятор расстояния/высоты символа

Формула: h = L·tg(α) или L = h / tg(α), где α — угловой размер символа (градусы).

  • Минимальная читаемость: α = 0.25° (14–16 угловых минут)
  • Оптимальные условия: α = 0.38° (20–22 угловых минут)
  • Критическая информация: α = 0.55° (30–35 угловых минут)

Пользователь может выбрать режим расчёта: точный (tg α) или приближённый (коэффициенты 0.004, 0.007, 0.009), как указано в исходной статье.

Зона оптимальной видимости (ЗОВ)

Позволяет задать координаты наблюдателей (глаз) в пространстве и рассчитать, какая часть строки символов (например, строка из 240 символов, занимающая всю ширину экрана) попадает в зону, где угловой размер символа превышает заданный порог. Результат — доля зоны оптимальной видимости для каждого наблюдателя и интегральный коэффициент эргономики Кэ (средняя доля × 10).

В расчёте учитываются размеры экрана (ширина, высота, кривизна), высота подвеса, положение строки по высоте. Это позволяет оценить проектные решения и при необходимости скорректировать расположение рабочих мест или размер экрана.

Электропитание и охлаждение

Суммарное энергопотребление оборудования влияет на выбор ИБП, сечение кабелей, количество и мощность распределительных щитов, а также на систему охлаждения.

Расчёт мощности

Общая активная мощность (Вт) суммируется по всем устройствам. Для ИБП необходимо учитывать коэффициент мощности (обычно 0.7–0.9) и выбирать полную мощность (ВА). Рекомендуемый запас — 20–30% от расчётной.

S (ВА) = (P_активная × 1.2) / cos φ

Тепловыделение

Тепловыделение (ккал/ч или кВт) примерно равно потребляемой мощности (Вт) с учётом КПД оборудования. Для LED-экранов тепловыделение составляет 30–40% от потребляемой мощности. Расчёт тепловой нагрузки необходим для подбора системы кондиционирования.

Q (ккал/ч) = P (Вт) × 0.86 × (1 - КПД)

Рекомендации по питанию

  • Для стоек с AV-оборудованием использовать отдельные линии питания, защищённые УЗО и автоматами.
  • Предусматривать возможность подключения к ИБП с автоматическим переключением (on-line).
  • Для LED-экранов большой площади использовать 3-фазное питание с равномерным распределением нагрузки.
  • Соблюдать требования ПУЭ по сечению кабелей в зависимости от тока и длины линии.

Настройка сетевого оборудования для AV-over-IP

Для корректной работы систем AV-over-IP (SDVoE, NDI, Dante AV) необходима правильная конфигурация коммутаторов. Ниже приведены примеры настроек для управляемых коммутаторов (на примере Cisco IOS).

Безопасность AV-систем

Современные AV-системы являются частью IT-инфраструктуры и должны защищаться от несанкционированного доступа, сетевых атак и физического вмешательства.

  • Сетевая сегментация: выделение AV-трафика в отдельные VLAN с запретом меж-VLAN-маршрутизации (если не требуется).
  • 802.1X (Network Access Control): аутентификация устройств перед предоставлением доступа к сети.
  • Шифрование потоков: для критичных данных (ВКС, управление) использовать SRTP (Secure RTP) или IPsec.
  • Управление доступом: на коммутаторах и матрицах использовать ACL (Access Control Lists) для ограничения управления только доверенными IP-адресами.
  • Парольная политика: смена заводских паролей, использование сложных паролей, централизованная аутентификация (RADIUS, TACACS+).
  • Физическая безопасность: стойки с замками, контроль доступа в серверные, защита от несанкционированного отключения кабелей.
  • Регулярное обновление ПО: установка обновлений прошивок для устранения уязвимостей.

Практические кейсы

Кейс 1. Ситуационный центр губернатора

Задача: спроектировать видеостену 4K и разместить 12 рабочих мест с оптимальной видимостью.

Решение: использован LED-калькулятор: шаг пикселя 1.5 мм, кабинеты 600×337.5 мм, 8 кабинетов по горизонтали и 3 по вертикали. Получено разрешение 7680×3240, физический размер 4.8×1.0125 м, площадь 4.86 м², мощность ~1.5 кВт. Эргономика: рассчитаны зоны видимости для 12 наблюдателей с учётом их расположения. Коэффициент эргономики Кэ = 8.9 (хорошо). Рекомендована установка экрана на высоте 1.2 м от пола, первые ряды на расстоянии 3.5 м, последние — 7 м.

Кейс 2. Конференц-зал на 50 мест

Задача: обеспечить качественное звукоусиление и разборчивость речи.

Решение: объём зала 450 м³, расчёт RT60 по Сэбину — исходное значение 1.5 с. За счёт установки акустических панелей (α=0.7) снижено до 0.7 с. Подобраны потолочные громкоговорители с чувствительностью 92 дБ/1Вт/1м, мощностью 30 Вт. SPL на расстоянии 8 м составил 92 + 10·log(30) - 20·log(8) ≈ 92 + 14.8 - 18.1 = 88.7 дБ, что достаточно для речевых программ. STI повысился с 0.52 до 0.71.

Кейс 3. LED-фасад здания

Задача: установка уличного LED-экрана площадью 120 м², шаг пикселя 16 мм, яркость 8000 нит.

Решение: выбраны кабинеты 960×960 мм (16×16 пикселей). Количество кабинетов: 12 по горизонтали, 10 по вертикали. Разрешение 192×160 пикселей? (неверно — исправлено: 12×16=192, 10×16=160, но это слишком низкое разрешение для рекламы). Правильно: для шага 16 мм и площади 120 м² размеры примерно 12×10 м, разрешение 750×625 пикселей. Мощность при 600 Вт/м² = 72 кВт. Подобран ИБП на 90 кВА с временем автономии 15 минут. Система охлаждения: тепловыделение 30% от мощности = 21.6 кВт, установлен сплит-система производительностью 24 кВт.

Часто задаваемые вопросы

Вопрос 1: Почему в калькуляторе видеотракта битрейт отличается от теоретического несжатого?

Ответ: В приложении используются эмпирические значения для сжатых потоков с учётом типовых профилей кодеков и накладных расходов протоколов. Для получения несжатого битрейта можно воспользоваться формулой в разделе «Методология».

Вопрос 2: Можно ли использовать калькулятор для расчёта наружных LED-экранов?

Ответ: Да. В режиме «По кабинетам» можно задать шаг пикселя и размеры кабинета, а также вручную ввести удельную мощность (для outdoor обычно 600–800 Вт/м²).

Вопрос 3: Как интерпретировать коэффициент эргономики Кэ?

Ответ: Кэ = (средняя доля зоны оптимальной видимости по всем наблюдателям) × 10. Значение 10 означает, что для всех участников 100% площади экрана попадает в зону с угловым размером символа не ниже критического. На практике Кэ > 7 считается хорошим, < 5 требует корректировки планировки.

Вопрос 4: Сохраняются ли данные при закрытии браузера?

Ответ: Да, все изменения автоматически сохраняются в localStorage. При очистке истории браузера данные могут быть удалены. Для надёжности используйте экспорт JSON.

Вопрос 5: Поддерживается ли работа в офлайн-режиме?

Ответ: Приложение работает полностью на стороне клиента, после загрузки страницы интернет-соединение не требуется (кроме загрузки шрифтов и иконок Font Awesome).

Вопрос 6: Можно ли добавить собственные типы кабинетов LED?

Ответ: Да, в режиме «По кабинетам» есть опция «Свой» для ввода пользовательских размеров и разрешения.

Вопрос 7: Как учесть акустические характеристики материалов помещения?

Ответ: В калькуляторе RT60 можно задать площади и коэффициенты поглощения для каждой поверхности (пол, стены, потолок, окна).

Управление проектом

Все данные, введённые пользователем, автоматически сохраняются в localStorage браузера. Для надёжности и переносимости предусмотрены следующие функции:

  • Сохранить в браузере — запись текущего состояния проекта в localStorage (выполняется автоматически при изменении параметров, также доступна ручная кнопка).
  • Экспорт JSON — выгрузка проекта в файл .json. Формат включает все разделы: видеотракты, LED-экраны, акустические настройки, тракты и пр.
  • Импорт JSON — загрузка ранее сохранённого проекта. При этом данные перезаписываются, интерфейс обновляется.
  • Печать отчёта — открывает новое окно с форматированным отчётом, содержащим все расчётные параметры и итоговые значения. Отчёт оптимизирован для печати (стили без интерактивных элементов).
  • Сброс проекта — удаление всех данных из localStorage и возврат к настройкам по умолчанию (требуется подтверждение).

Важно: данные не синхронизируются между устройствами. Для переноса проекта используйте экспорт/импорт JSON. Рекомендуется периодически делать резервные копии.
