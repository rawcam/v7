Общее описание

Sputnik Studio v7.0 — веб-приложение для инженерного проектирования аудиовизуальных систем и управления проектами. Это полностью переработанная версия, которая включает:

  • Все инженерные расчёты (видео, сеть, LED, звук, ВКС, эргономика, питание), перенесённые из предыдущей версии 6.x.
  • Новую модульную архитектуру с единым состоянием и подписками.
  • Топбар с переключением разделов: Главная, Проекты, Расчёты, Шаблоны.
  • Дашборд с виджетами (общая статистика, бюджет, прогресс, срочные проекты, встречи).
  • Систему управления проектами (создание, редактирование, дорожная карта, встречи, закупки).
  • Горизонтальную карусель активных проектов.
  • Редактирование сайдбара (drag-and-drop).
  • Логгер с парольной защитой.

Старая версия (6.x) с традиционным сайдбаром и калькуляторами доступна по ссылке: Sputnik Studio v6.1.0.

Руководство пользователя

Главная страница (Дашборд)

Дашборд содержит:

  • Общую статистику – количество проектов по статусам (всего, активные, ожидают оплату, стадия П, стадия Р, монтаж, срочные, в архиве).
  • Виджеты – общий бюджет активных проектов, средний прогресс, список срочных проектов, общее количество встреч и три ближайшие.
  • Карусель активных проектов – горизонтальная прокрутка карточек проектов. Кнопки навигации вынесены за пределы контента, но остаются в видимой области.

Раздел «Проекты»

Список проектов в виде карточек. Клик по карточке или кнопка «Редактировать» открывает детальную страницу проекта, где можно:

  • Редактировать название, статус, приоритет, бюджет, дату начала, инженера, руководителя проекта.
  • Прогресс рассчитывается автоматически в зависимости от статуса (10%, 30%, 60%, 80%, 100%).
  • Управлять дорожной картой (текущая и следующая стадия с датами).
  • Добавлять/удалять встречи и закупки с расширенными статусами (ожидает оплаты, оплачено, зарезервировано, заказано, в пути, доставлено, отменено, нет в наличии).
  • Сохранять изменения или удалить проект.

Раздел «Расчёты»

Здесь доступны все инженерные калькуляторы, перенесённые из версии 6.x: видео, сеть, LED, звук (SPL, падение, изменение, RT60, подбор громкоговорителей), ВКС (P2P и многоточечный), эргономика, питание (ИБП + кондиционер). Сайдбар содержит аккордеоны для настройки параметров. Аккордеоны можно перетаскивать (режим редактирования сайдбара).

Раздел «Шаблоны»

Конструктор документов позволяет выбрать шаблон (пояснительная записка, спецификация, акт), отметить подсистемы проекта и привязать документ к проекту. Сгенерированный текст можно скопировать.

Модульная архитектура

Все модули используют единое состояние AppState (паттерн «наблюдатель»). Изменения состояния инициируются только в обработчиках событий, а подписчики (модули) обновляют DOM. Это исключает рекурсивные вызовы и обеспечивает предсказуемость.

Стили (styles.css)

Используются CSS-переменные для поддержки светлой и тёмной темы. Основные параметры:

  • Шрифт: font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif;
  • Базовый размер: 14px, межстрочный интервал 1.5.
  • Цветовая схема: градиентный фон, полупрозрачные панели (backdrop-filter: blur(12px)), акцентный цвет #2563eb (светлая тема) / #3b82f6 (тёмная).
  • Анимации: плавные переходы (transition: all 0.25s cubic-bezier(0.2, 0.9, 0.4, 1.1)), анимация появления fadeInUp.
  • Карточки: скругление 32px, тень, при наведении подъём.
  • Карусель: реализована на Swiper, кнопки навигации вынесены за пределы контента (относительное позиционирование родителя).

Расчёты и формулы

Все расчёты сосредоточены в модулях, используют функции из utils.js. Основные формулы:

Битрейт видео = база (300/1000/4000) × коэффициент субдискретизации × коэффициент цветового пространства × коэффициент битности × (FPS/60)
SPL = чувствительность + 10·lg(мощность) – 20·lg(расстояние) + headroom + помещение
RT60 = 0.161 × V / Σ(αᵢ·Sᵢ)
LED-разрешение = (ширина_кабинета_мм / шаг_мм) округление вниз × кол-во_кабинетов
Прогресс проекта = 10% (пресейл), 30% (стадия П), 60% (стадия Р), 80% (монтаж), 100% (завершён)

Хранение данных

  • localStorage: проекты, настройки видео/сети, порядок аккордеонов, тема, имя проекта, логи.
  • Индексы: ключи sputnik_projects, sputnik_studio_project, sidebar_order, sputnik_logs и др.
  • При сбросе проекта данные удаляются из localStorage и страница перезагружается.

Карусель активных проектов

Реализована с помощью библиотеки Swiper. Кнопки навигации вынесены за пределы контейнера с контентом (.dashboard-wrapper) с помощью обёртки .projects-swiper-wrapper и отрицательных отступов. Это позволяет кнопкам быть видимыми, не нарушая центрирование.

<div class="projects-swiper-wrapper">
    <div class="swiper-container">
        <div class="swiper-wrapper">...</div>
        <div class="swiper-button-prev"></div>
        <div class="swiper-button-next"></div>
        <div class="swiper-pagination"></div>
    </div>
</div>

Логгер

Модуль logger.js записывает действия пользователя, изменения состояния, информацию о сети и системе. Доступ к логам защищён паролем (генерируется при первом запуске, выводится в консоль). Логи хранятся в localStorage, их можно просмотреть через кнопку «Показать логи» в разделе «Управление».

Глоссарий

SPL — уровень звукового давления, дБ.

RT60 — время реверберации, с.

PoE — передача питания по Ethernet (802.3af/at/bt).

QoS — качество обслуживания (приоритизация трафика).

Multicast — групповая рассылка.

Chroma Subsampling — цветовая субдискретизация (4:4:4, 4:2:2, 4:2:0).

Bit Depth — глубина цвета (8/10/12 бит).

MCU/SFU — многоточечные устройства для видеоконференций.

Latency — задержка, мс.

PTP — протокол точного времени IEEE 1588.
